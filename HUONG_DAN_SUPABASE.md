# HƯỚNG DẪN CẤU HÌNH VÀ VẬN HÀNH SUPABASE (HYBRID BACKEND)

Tài liệu này hướng dẫn chi tiết cách thiết lập dự án Supabase, tạo các bảng cơ sở dữ liệu, phân quyền và cấu hình đồng bộ hóa từ **Google Sheets** sang **Supabase** cho ứng dụng **Zphysics**.

---

## BƯỚC 1: TẠO DỰ ÁN SUPABASE (MIỄN PHÍ)

1. Truy cập vào [Supabase](https://supabase.com) và đăng ký tài khoản (bằng GitHub hoặc Email).
2. Bấm vào **New Project** để tạo một dự án mới:
   - **Name**: Nhập `Zphysics` (hoặc tên tùy chọn).
   - **Database Password**: Nhập mật khẩu cơ sở dữ liệu của bạn (hãy lưu lại mật khẩu này).
   - **Region**: Chọn khu vực gần Việt Nam nhất (ví dụ: `Singapore - ap-southeast-1`).
   - **Pricing**: Chọn gói **Free** (Miễn phí).
3. Bấm **Create new project** và đợi khoảng 1-2 phút để Supabase chuẩn bị môi trường.

---

## BƯỚC 2: TẠO CÁC BẢNG DỮ LIỆU & PHÂN QUYỀN

1. Tại thanh menu bên trái của bảng quản trị Supabase, chọn biểu tượng **SQL Editor** (hình thẻ mã nguồn `SQL`).
2. Bấm vào **New query** (hoặc tạo một tab SQL mới).
3. Copy toàn bộ đoạn mã SQL dưới đây và dán vào ô soạn thảo:

```sql
-- 1. BẢNG LƯU TRỮ CÂU HỎI
create table questions (
  id bigint generated by default as identity primary key,
  made text not null,                -- Mã đề (Ví dụ: 12_c1_b03)
  question_id int not null,          -- ID câu hỏi trong mã đề (1, 2, 3...)
  question text not null,            -- Nội dung câu hỏi
  a text,                            -- Lựa chọn A
  b text,                            -- Lựa chọn B
  c text,                            -- Lựa chọn C
  d text,                            -- Lựa chọn D
  correct text,                      -- Đáp án đúng (A, B, C hoặc D)
  explanation text,                  -- Hướng dẫn giải chi tiết
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  
  -- Tạo ràng buộc để đảm bảo một mã đề không trùng lặp số câu hỏi (hỗ trợ Upsert nhanh)
  constraint unique_made_question_id unique (made, question_id)
);
create index idx_questions_made on questions(made);
create index idx_questions_made_qid on questions(made, question_id);

-- 2. BẢNG LƯU TRỮ HỌC SINH
create table students (
  email text primary key,            -- Gmail học sinh
  full_name text,                    -- Tên học sinh
  lop text,                          -- Lớp học sinh (10, 11, 12,...)
  role text default 'free',          -- 'free' hoặc 'premium'
  premium_until timestamp with time zone, -- Hạn Premium (null = trọn đời / vô hạn)
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. BẢNG LƯU TRỮ LINK BÀI HỌC
create table lessons (
  id bigint generated by default as identity primary key,
  grade text,                        -- Khối lớp (10, 11, 12)
  title text not null,               -- Tên bài học
  video_url text,                    -- Link video bài giảng
  exercise_url text,                 -- Link bài tập
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 4. BẢNG DANH SÁCH ĐỀ THI LUYỆN ĐỀ VIP
create table exams (
  id text primary key,               -- Mã đề thi
  title text not null,               -- Tiêu đề đề thi
  description text,                  -- Mô tả ngắn
  pdf_url text,                      -- Link tải file PDF đề thi
  mc_answers text,                   -- Đáp án trắc nghiệm nhiều lựa chọn
  tf_answers text,                   -- Đáp án Đúng/Sai
  sa_answers text,                   -- Đáp án điền ngắn
  duration integer default 50,       -- Thời gian làm bài (phút)
  grade text default '12',           -- Lớp
  video_url text,                    -- Link video chữa bài
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 5. PHÂN QUYỀN ĐỌC CÔNG KHAI (SELECT) CHO HỌC SINH QUA KEY "ANON"
alter table questions enable row level security;
create policy "Allow public read questions" on questions for select using (true);

alter table students enable row level security;
create policy "Allow public read students" on students for select using (true);

alter table lessons enable row level security;
create policy "Allow public read lessons" on lessons for select using (true);

alter table exams enable row level security;
create policy "Allow public read exams" on exams for select using (true);

-- 6. BẢNG TIẾN TRÌNH HỌC TẬP (STUDENT PROGRESS)
create table student_progress (
  id bigint generated by default as identity primary key,
  email text references students(email) on delete cascade,
  lesson_id text not null,       -- ID bài học (ví dụ: 'cauhoi_12_c1_b03') hoặc mã đề thi (ví dụ: 'de_thi_thu_01')
  type text not null,            -- 'quiz', 'exam', 'video', 'document'
  score numeric,                 -- Điểm số nếu có
  completed_at timestamp with time zone default timezone('utc'::text, now()) not null,
  constraint unique_student_lesson_progress unique (email, lesson_id, type)
);

alter table student_progress enable row level security;
create policy "Allow student to read own progress" on student_progress for select using (auth.role() = 'anon' or true);
create policy "Allow student to insert own progress" on student_progress for insert with check (true);
create policy "Allow student to update own progress" on student_progress for update using (true);

create index idx_progress_email on student_progress(email);
create index idx_progress_email_lesson on student_progress(email, lesson_id);
```

4. Bấm nút **Run** (phía dưới bên phải màn hình). Xác nhận kết quả hiển thị thông báo chạy thành công (`Success. No rows returned` hoặc tương tự).

## BƯỚC 3: LẤY THÔNG TIN CẤU HÌNH DỰ ÁN

Tại Supabase, bấm vào biểu tượng bánh răng **Project Settings** ở góc dưới cùng bên trái màn hình -> Chọn mục **API**:
1. Copy giá trị tại mục **Project URL** (Ví dụ: `https://abcdefghijkl.supabase.co`).
2. Tại mục **Project API keys**:
   - Copy key **`anon public`** (Dùng để thay vào các file HTML của học sinh).
   - Copy key **`service_role secret`** (Bấm *Reveal* để hiển thị. **Lưu ý quan trọng**: Khóa này có quyền admin toàn quyền ghi đè RLS, chỉ được điền vào Google Sheets Apps Script, tuyệt đối không lộ ra ngoài).

---

## BƯỚC 4: CẤU HÌNH VÀ KÍCH HOẠT TRÊN GOOGLE SHEETS

1. Mở file Google Sheet quản lý đề thi của bạn.
2. Chọn **Tiện ích mở rộng (Extensions)** -> **Apps Script**.
3. Mở file mã nguồn `QuizAPI.gs`.
4. Tìm đến đoạn cấu hình Supabase ở đầu file (khoảng dòng 30):
   ```javascript
   // =========================================================================
   // CẤU HÌNH SUPABASE (Hãy điền thông tin dự án Supabase của bạn tại đây)
   // =========================================================================
   var SUPABASE_URL = "DÁN_PROJECT_URL_CỦA_BẠN_VÀO_ĐÂY";
   var SUPABASE_KEY = "DÁN_SERVICE_ROLE_KEY_CỦA_BẠN_VÀO_ĐÂY";
   // =========================================================================
   ```
   Dán URL và Key (`service_role` key) tương ứng bạn vừa lấy ở Bước 3 vào đây.
5. Bấm biểu tượng **Save** (Hình đĩa mềm) để lưu lại code.
6. **Deploy lại Web App**:
   - Chọn nút **Deploy** (Góc trên bên phải) -> **Manage Deployments**.
   - Bấm vào biểu tượng bút chì để sửa (Edit) phiên bản hiện tại, chọn **New version** (Phiên bản mới).
   - Bấm **Deploy** để kích hoạt lại Web App.

---

## BƯỚC 5: ĐỒNG BỘ DỮ LIỆU LẦN ĐẦU (BULK SYNC)

1. Quay lại trang Google Sheet của bạn và **F5 tải lại trang**.
2. Đợi vài giây, trên thanh công cụ của Google Sheet (ở cạnh Menu Help/Trợ giúp) sẽ xuất hiện một mục menu mới tên là **`Zphysics Tools`**.
3. Bấm vào **`Zphysics Tools`** -> Chọn **`Đồng bộ tất cả dữ liệu sang Supabase`**.
4. Sẽ có hộp thoại xác nhận hiện ra. Bấm **YES** để bắt đầu đồng bộ.
5. Sau khi đồng bộ thành công, sẽ có hộp thoại báo **Thành công**. Bạn có thể quay lại Supabase Studio, vào mục **Table Editor** để xem dữ liệu đã được đẩy lên đầy đủ ở cả 4 bảng.

## BƯỚC 6: CẤU HÌNH CHO APP HỌC SINH (FRONTEND)

Trước khi đóng gói app (Android/iOS) hoặc deploy lên Netlify, bạn cần mở mã nguồn trên máy tính và cập nhật cấu hình API Supabase ở các file sau:

1. **`www/index.html`**:
   - Dòng 1159: Thay thế `SUPABASE_URL` và `SUPABASE_ANON_KEY` bằng khóa **`anon public`** của bạn.
2. **`www/tracnghiem.html`**:
   - Dòng 1014: Thay thế `SUPABASE_URL` và `SUPABASE_ANON_KEY` bằng khóa **`anon public`** của bạn.
3. **`www/luyende.html`**:
   - Dòng 856: Thay thế `SUPABASE_URL` và `SUPABASE_ANON_KEY` bằng khóa **`anon public`** của bạn.

## CƠ CHẾ VẬN HÀNH SAU KHI CẤU HÌNH XONG

- **Khi học sinh vào App**: App sẽ gọi thẳng qua API Supabase bằng `anon` key để load đề thi/học sinh chỉ trong vòng **0.05 giây** (trước đây là 2-3 giây).
- **Khi Giáo viên chỉnh sửa trên Google Sheet**:
  - Giáo viên chỉ việc sửa trực tiếp trên file Google Sheet bình thường. Sau khi sửa xong, bấm **`Zphysics Tools` -> `Đồng bộ tất cả dữ liệu sang Supabase`** để cập nhật dữ liệu mới lên App.
